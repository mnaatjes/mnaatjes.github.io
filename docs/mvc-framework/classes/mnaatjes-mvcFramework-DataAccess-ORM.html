ORM.php : 52

ORM (Object-Relational Mapper)

This class provides a simplified interface for interacting with a relational database. It abstracts common database operations such as C-R-U-D (Create, Read, Update, Delete) and raw query execution using PDO prepared statements to ensure security and prevent SQL injection.

Counts the number of rows in a specified table that match the given conditions.

public count(string $table_name[, array<string|int, mixed> $conditions = [] ]) : int|bool

This method constructs a SELECT COUNT(*) query and safely executes it using prepared statements. It supports different formats for specifying conditions in the $conditions array.

Parameters
$table_name : string

The name of the table to count rows from.

$conditions : array<string|int, mixed> = []

An array of conditions to filter the count. Conditions can be:

  • An associative array: ['column_name' => 'value'].
  • An indexed array with two elements: ['column_name', 'value'] (defaults to = operator).
  • An indexed array with three elements: ['column_name', 'operator', 'value'].
  • An array of the above formats to apply multiple conditions with AND.

Executes a SELECT query with one or more JOIN clauses, allowing for custom selection, ordering, and limiting.

This method constructs a SQL query from a primary table and an array of join clauses. It supports two formats for defining join clauses: a simple string format or an associative array.

Parameters
$primaryTable : string

The name of the main table for the query.

$joins : array<string|int, mixed> = []

An array of join clauses. Each element can be:

  • A string (e.g., "users INNER JOIN posts ON posts.user_id = users.id").
  • An associative array (e.g., ["table" => "posts", "on" => "posts.user_id = users.id"]). The key should be the join keyword (e.g., 'on') and the value the condition.
$selects : array<string|int, mixed> = ["*"]

An array of columns to select from the tables. Defaults to ["*"].

$conditions : array<string|int, mixed> = []

An array of conditions to filter the count. Conditions can be:

  • An associative array: ['column_name' => 'value'].
  • An indexed array with two elements: ['column_name', 'value'] (defaults to = operator).
  • An indexed array with three elements: ['column_name', 'operator', 'value'].
  • An array of the above formats to apply multiple conditions with AND.
$orderBy : array<string|int, mixed> = []

An associative array for ordering the results (e.g., ["column_name" => "DESC"]).

$limit : int = 0

The maximum number of rows to return. A value of 0 means no limit.

$fetchMethod : string = "fetchAll"

The PDO fetch method to use (e.g., fetchAll, fetch).

$fetchStyle : int = PDO::FETCH_ASSOC

The PDO fetch style constant (e.g., PDO::FETCH_ASSOC).

query()

ORM.php : 499

Query Methods: Raw Query

public query(string $sql[, array<string|int, mixed> $bindings = [] ][, string $fetchMethod = "fetchAll" ][, int $fetchStyle = PDO::FETCH_ASSOC ]) : mixed

Executes a SQL query with prepared statements and parameter bindings.

This method is the core execution logic for the ORM. It handles SELECT, INSERT, UPDATE, and DELETE queries, and returns the appropriate result based on the query type. It uses PDO prepared statements to prevent SQL injection attacks.

Parameters
$sql : string

The SQL query string to be executed.

$bindings : array<string|int, mixed> = []

An associative or indexed array of values to bind to the prepared statement. Defaults to an empty array.

$fetchMethod : string = "fetchAll"

The PDO fetch method to use for SELECT queries (e.g., "fetchAll", "fetch").

$fetchStyle : int = PDO::FETCH_ASSOC

The PDO fetch style constant (e.g., PDO::FETCH_ASSOC) to be used with the fetch method.

Return values
mixed —

The result of the query:

  • For SELECT queries: An array of rows or a single row, depending on the fetch method.
  • For INSERT queries: The ID of the last inserted row.
  • For UPDATE or DELETE queries: The number of affected rows.
  • On failure: false.
